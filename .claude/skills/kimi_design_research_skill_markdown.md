# Skill: Kimi Design Style Research

## Skill ID

```text
kimi-design-style-research
```

---

# Purpose

该 Skill 用于：

```text
分析 Kimi / Moonshot AI 的产品设计风格、视觉语言、交互哲学、品牌人格、功能组织方式、以及 Agent 工作台产品形态。
```

并输出：

```text
产品设计调研报告
AI Agent 产品设计参考
竞品设计对比
设计语言拆解
UI/UX 风格分析
品牌人格分析
产品信息架构分析
```

适用于：

- AI Agent 平台
- AI IDE
- AI 工作流产品
- AI SaaS
- AgentStudio
- 多 Agent 系统
- 工业 AI 平台
- AI 办公产品

---

# Core Objective

你需要像：

```text
高级 AI 产品设计研究员
+ SaaS 产品设计负责人
+ AI Agent 平台架构顾问
```

一样工作。

目标不是：

```text
简单评价“好不好看”
```

而是：

```text
分析设计背后的产品哲学、用户定位、交互模型、品牌策略、Agent 工作流组织方式。
```

---

# Analysis Dimensions

必须严格从以下维度分析。

禁止遗漏。

---

# 1. Brand Positioning

分析：

```text
Kimi 的品牌定位
Moonshot AI 的产品气质
目标用户
品牌人格
产品情绪
品牌关键词
```

输出示例：

```text
年轻化
轻量
工具化
AI 工作台
非传统聊天机器人
弱企业感
强个人生产力
```

必须分析：

- 为什么 Kimi 采用这种品牌人格
- 为什么没有走“企业后台”风格
- 为什么强调轻量感
- 为什么保留 doodle / 宇宙 / 月亮元素

---

# 2. Visual Language

分析：

```text
视觉语言
配色体系
圆角体系
间距体系
字体倾向
留白策略
图标风格
阴影层级
卡片系统
```

必须分析：

## Color Strategy

例如：

```text
低饱和
柔和辅助色
弱品牌主色
白底黑字
```

分析其原因：

```text
降低 AI 压迫感
提高长时间使用舒适度
强调工具属性
```

---

## Typography

分析：

```text
标题层级
正文层级
输入框字体
信息密度
```

---

## Spacing

分析：

```text
为什么大量留白
为什么组件间距较大
为什么避免复杂边框
```

---

# 3. Interaction Philosophy

分析：

```text
交互哲学
用户路径
任务流
Agent 调度入口
AI 工作流设计
```

重点分析：

## Chat-first Architecture

Kimi 的核心：

```text
一个输入框
→ 多工具系统
→ 多 Agent
→ 多产物生成
```

必须分析：

- 为什么仍然保留聊天入口
- 为什么功能入口越来越显性化
- 为什么开始弱化“纯聊天”
- 为什么强调“任务完成”

---

## Result-oriented UX

分析：

```text
Slides
Docs
Websites
Research
Sheets
```

为什么：

```text
产物导向
结果导向
工作流导向
```

而不是：

```text
仅仅聊天
```

---

# 4. Information Architecture

分析：

```text
产品信息架构
导航结构
Agent 能力组织方式
模块拆分
```

必须输出：

## Functional Layering

例如：

```text
基础聊天层
工具调用层
Agent 工作流层
内容生成层
资产管理层
```

---

## Why Explicit Tools Matter

必须分析：

为什么 Kimi 不把所有能力隐藏在 Prompt 后面。

例如：

```text
Slides
Websites
Research
```

为什么必须做成显式入口。

---

# 5. AI Product Design Philosophy

分析：

```text
Kimi 的 AI 产品哲学
```

例如：

```text
AI 不应该只是聊天
AI 应该是任务执行器
AI 应该生成结果
AI 应该变成工作台
```

必须分析：

- 为什么 AI 产品正在从 Chat 转向 Workspace
- 为什么 Agent UI 正在替代传统 Chat UI
- 为什么工具入口越来越重要
- 为什么 AI IDE 风格正在形成

---

# 6. Competitor Comparison

必须与以下产品比较：

```text
ChatGPT
Claude
DeepSeek
豆包
通义千问
Cursor
Lovable
Manus
```

分析：

```text
视觉差异
产品差异
Agent 组织方式
工作流差异
品牌人格差异
```

禁止只写一句话。

必须深入分析：

```text
为什么形成这种差异
背后的用户定位是什么
```

---

# 7. Agent Product Inspiration

这一部分最重要。

你必须输出：

```text
Kimi 对 AI Agent 平台的启发
```

例如：

## Agent UI

```text
不要只有聊天框
```

而应该：

```text
任务入口
能力卡片
工具工作台
Agent 状态
执行流
产物区
```

---

## Agent Runtime UX

分析：

```text
为什么用户需要看到：

执行过程
工具调用
状态变化
结果生成
```

---

## Workspace-first Design

分析：

为什么未来 AI 产品会变成：

```text
AI Workspace
```

而不是：

```text
Chat App
```

---

# 8. Industrial AI Adaptation

如果目标产品是：

```text
工业 AI
工业 Agent
工业自动化平台
```

必须继续分析：

## 如何工业化迁移 Kimi 风格

包括：

```text
工业控制台
工业工作流
设备状态
工厂监控
Agent 调度
任务执行
```

必须分析：

```text
哪些设计语言能保留
哪些必须更专业化
哪些不能过于娱乐化
```

---

输出必须：

```text
结构化
专业化
高信息密度
产品级
设计评审级
```

禁止：

```text
空话

```

```markdown
# Brand Positioning

# Competitor Comparison

# Industrial AI Adaptation

# Final Conclusion
```

---

# Tone

语气必须：

```text
专业
克制
像产品设计评审文档
像 AI 产品战略分析
```

禁止：

```text
水文
口语化
情绪化
```

---

# Important Constraint

你的目标不是：

```text
模仿 Kimi 的 UI
```

而是：

```text
理解 Kimi 为什么这样设计
```

必须始终分析：

```text
设计背后的 Agent 产品趋势
AI 工作流趋势
AI Workspace 趋势
```

而不是停留在：

```text
颜色
按钮
圆角
```

层面。
